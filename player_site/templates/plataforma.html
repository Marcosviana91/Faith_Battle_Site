Sobre o Projeto

Este site faz parte de um projeto individual e não tem vínculo com o jogo oficial ou seus criadores. O objetivo deste site é divulgar o andamento do desenvolvimento do jogo digital não oficial. Gosto de dizer que ainda está em alpha e será constantemente desenvolvido.

A meta é fomentar a comunidade do jogo físico.

Este projeto consiste em 4 partes:

  • Front-End: aplicativo móvel para Android.
  • Back-End: servidor do jogo.
  • Back-End: servidor de autenticação e site.
  • Back-End: Bancos de dados (Postgre e Mongo).

Back-End's dockerizados individualmente.

Front-End: Aplicativo móvel

Um aplicativo simples desenvolvido em React Native, porém suficiente para jogar. O design, animações, tutorial, UI/UX foram deixados para um outro momento. Mesmo assim vem sofrendo mudanças conforme o desenvolvimento avança.

Back-End: Servidor do Jogo

API em Python utilizando FastAPI, responsável por tratar as regras do jogo, como comparação de ataque e defesa entre heróis durante um combate, adição de efeitos de um milagre ou pecados e itens. Também é responsável pelo gerenciamento de salas, jogadores conectados e confirmação de tokens de authenticação.

Back-End: Servidor de autenticação e site.

Plataforma web desenvolvida em Django (Python), responsável por tratar authenticação de jogadores através da consulta em banco de dados. Ela se comunica com a API citada acima.

Através desta, será possível o jogador editar seus decks, analizar suas partidas e assistir outras partidas.

Back-End: Bancos de dados (Postgre e Mongo).

Um simples docker rodando uma imagem do PostgreSQL e outro do Mongo db. Por falta de compatibilidade com o hardware do servidor, o Mongo foi substituído pelo TinyDB, um banco de dados minimalista, assim como o sqlite, porém em pythyon e orientado a documentos.

Os bancos armazenam dados do usuário e estatísticas do jogador, como XP, deck build, conquistas, artes desbloqueadas, etc
